JavaWeb开发runtimeEnvironment相关

  • 时间：2017-03-17
    • JavaWeb开发

最近用spring3.2+mybatis3.2框架写了个demo，一运行报错：

java.lang.IllegalArgumentException

  at …

   …

上stackoverflow一查，so easy,spring与jdk版本的问题。俩选择：要么用spring4+jdk1.8,要么用降jdk为1.7.

降jdk！系统切换jdk环境修改bashrc就可。

那么问题又来了！eclipse上tomcat又运行报错：

The JRE could not be found. Edit the server and change the JRE location

又是一通Google。同样so easy！

换掉Project的JRE后，

  • Go to Eclipse -> Preference
  • Search for runtime
  • Click on Runtime Environments
  • Select Apache Tomcat 8 -> click Edit

一切迎刃而解，如春风拂面舒适。

再回头看，不由得深思！每每报错，程序已经很明显的把错误指出了。其实只需要去看错误信息即可立马去改正，根本无需去Google去百度！心烦意燥是开发过程中的大忌，该改之，勇敢去面对错误提示，静下心去思考，而不是动辄上网找答案。
